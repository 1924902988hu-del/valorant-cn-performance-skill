# 服别、反作弊与虚拟化矩阵

## 先确认服别

| 情况 | 反作弊线索 | 安全动作 |
|---|---|---|
| 国服 | 腾讯客户端、ACE/SGuard 进程可能出现 | 仅按腾讯官方的具体弹框路径排查 |
| 国际服/港服 | Riot Client、Vanguard `vgc`/`vgk` 服务 | 保持 Secure Boot/TPM；区分传统 Vanguard 与可选 On-Demand |
| 不确定 | 进程/服务只能作线索，不能绝对判定 | 追问安装来源与登录客户端；只做跨服低风险检查 |

诊断脚本的反作弊发现是启发式：服务未运行不等于没安装，进程出现也不证明当前账号服别。

## 国服 ACE：只处理特定弹框

腾讯 2026-01-04 官方指引针对的是：**“未开启或有其他软件占用 CPU 虚拟化功能”**。

正确顺序：

1. 获取弹框原文或截图，确认就是该问题。
2. 根据主板/整机厂商文档确认 BIOS 中 CPU 虚拟化与 IOMMU/VT-d 状态。BIOS 名称因厂商而异，不盲写菜单路径。
3. 只有 BIOS 已按官方要求设置、弹框仍存在时，才将 HVCI、VBS、Hyper-V、WSL2、Sandbox、虚拟机等列为冲突候选。
4. 在任何关闭建议前说明：HVCI/VBS 是 Windows 内核安全保护；Hyper-V/WSL2/Docker/虚拟机可能依赖虚拟化；Windows Hello/企业策略也可能受影响。
5. 优先给出腾讯客服/官方文档路径。若用户明确接受风险并要继续，逐项记录原值、单项测试、重启复测并准备回滚。

这不是“国服必须永久关闭内存完整性/Hyper-V”的一般规则，也不是 FPS 优化项。

## 国际服/港服 Vanguard

- 不建议关闭 Secure Boot 或 TPM。错误码必须按 Riot 当前支持文档处理。
- Riot 2026-06-24 公布的 **Vanguard On-Demand / Pre-Check 是可选项**。希望使用它的机器至少需要 Windows 11 25H2、UEFI Secure Boot、TPM 2.0、VBS、HVCI、IOMMU 等前置条件。
- 因而“为帧数关闭 VBS/HVCI”不仅降低 Windows 安全保护，还会失去 On-Demand 资格。必须先问用户是否使用/计划使用该模式。
- 未使用 On-Demand 也不代表关闭安全能力有确定性能收益；只有可重复基准能支持个体结论。

## 红线

- 不结束、禁用或删除反作弊服务来玩游戏。
- 不加载测试签名驱动，不关闭驱动签名强制，不修改内核或反作弊文件。
- 不使用 ACE-KILLER、反作弊屏蔽器、进程隐藏或硬件身份伪装。
- 不承诺任何改动“绝不封号”；合法配置问题仍以产品方支持为准。
