# 症状路由与设置判断

## 1. 客户端 FPS 路径

信号：FPS 与帧时间同时恶化；1% low 明显低于平均帧；降分辨率/画质或减少后台负载会改变结果。

先查：

- Windows 当前刷新率、分辨率与目标显示器。
- 游戏是否使用目标 GPU；笔记本是否接电、是否处于厂商性能模式、是否受温度/功耗限制。
- CPU/GPU/内存/显存占用与温度；后台录制、浏览器、覆盖层是否在同一时刻抢占。
- 同一驱动/同一补丁/同一场景下的可重复基线。

不要从 WMI 的内存频率直接断言 XMP/EXPO 或双通道已开启；需要 BIOS/厂商工具或更可靠的内存信息确认。

## 2. 1% low / 瞬时卡顿路径

固定同一场景采集帧时间。先排查：

- 温度或功耗撞墙。
- 内存/显存压力与分页。
- 同步录制、覆盖层、更新、杀毒扫描等时间相关后台任务。
- 新补丁/新驱动引入的回归。

“ACE 扫盘导致卡顿”等社区说法只能作为时间相关假设；没有腾讯一手确认或本机 A/B 数据时，不指示关闭安全软件或反作弊。

## 3. 高 FPS 但不跟手

- 确认 Windows 的实际刷新率，不只看显示器包装参数。
- 对 NVIDIA 兼容显卡，优先在游戏内评估 Reflex；不要同时叠加多个来源不明的低延迟开关。
- 对兼容 AMD 硬件与 2026-03-09 以后所需驱动，评估游戏内 Anti-Lag 2。
- 检查垂直同步、帧率上限和显示模式，但每项用同一基线 A/B 测试；不存在所有机器通用的唯一数值。
- Raw Input Buffer 已在 11.06 移除并常开，不再让用户寻找该选项。

## 4. 网络假卡顿

信号：FPS/帧时间稳定，但 RTT、丢包、角色回弹或命中反馈异常。

区分实验：

- 同一时间记录游戏内网络图表与帧时间。
- 有条件时用有线网络与手机热点做短时对照；热点只用于定位，不保证更优。
- 不把 ping 下降等同于 FPS 上升，也不把加速器营销数据当本机证据。
- 不收集公开 IP、MAC、账号令牌或完整日志中的身份信息。

## 5. 当前版本设置事实

- 11.02（2025-07-29）：引擎从 Unreal 4.27 升到 Unreal 5.3。
- 11.06（2025-09-16）：Raw Input Buffer 开关移除，始终启用。
- 12.09（2026-05-12）：加入 AMD Anti-Lag 2；仅适用于兼容硬件与规定日期后的驱动。
- NVIDIA Reflex 是 VALORANT 官方支持路径；效果随系统是否 GPU-bound 等条件变化。

对画质、分辨率、帧率上限、HAGS、Game Mode、驱动面板设置都使用实测，不写“必开/必关”。
