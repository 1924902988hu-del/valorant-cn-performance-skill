# 诊断方案

## 结论与置信度

最可能瓶颈不是 RTX 4060 或 220 FPS，而是屏幕实际仅运行在 **60Hz**；这是典型的“高 FPS 但不跟手”显示/输入链路问题。**置信度：高。**

220 FPS 约每 **4.55ms** 生成一帧，但 60Hz 屏幕约每 **16.67ms** 才刷新一次；切换到 240Hz 后，刷新间隔约为 **4.17ms**。这通常会带来最明显的跟手改善。

## 已知 / 缺失

- 用户确认：国服游戏本、RTX 4060、游戏约 220 FPS、接电、屏幕标称 240Hz、Windows 当前为 60Hz。
- 当前判断：显示刷新率是首要矛盾。
- 未运行诊断脚本，也没有修改任何设置。
- 仍缺：CPU、1% low、帧时间、温度、游戏显示模式、垂直同步和 NVIDIA Reflex 状态，以及 Windows 是否提供 240Hz 选项。

## 症状分类

主要属于：**显示与输入延迟路径**。

如果切换到 240Hz 后仍有明显卡顿，再区分：

- FPS/帧时间突然恶化：客户端性能或温度、功耗问题。
- FPS/帧时间稳定，但角色回弹、命中反馈延后：网络路径。

## 优先动作

1. **先把 Windows 实际刷新率改为 240Hz**
   - 路径：`设置 → 系统 → 屏幕 → 高级显示`。
   - 先确认顶部选中的是游戏本内置屏幕，再将“选择刷新率”改为 **240Hz**。
   - 改完重新进入该页面，确认当前值确实显示 240Hz；不能只看屏幕宣传规格。
   - 如果有“动态 60/240Hz”，第一轮测试先固定 240Hz。
   - 依据等级：官方/一手，Microsoft。
   - 适用条件：下拉菜单存在 240Hz。
   - 预期观察：鼠标、桌面拖动和游戏转视角都会立刻更顺、更跟手。
   - 回滚：在同一位置恢复 60Hz。

2. **如果只有 60Hz，不要强制创建自定义 240Hz**
   - 先检查是否选错显示器、当前是否使用屏幕原生分辨率，以及厂商控制中心是否处于省电/低刷新率模式。
   - 然后根据游戏本准确型号检查厂商显示驱动和面板模式。驱动安装、MUX 独显直连切换或 BIOS 操作应另行记录原值并单独测试。
   - 依据等级：本机状态检查；具体支持能力需要整机厂商一手资料确认。
   - 适用条件：Windows 没有提供 240Hz。
   - 预期观察：找到并恢复面板原生 240Hz 选项。
   - 回滚：恢复厂商控制中心原模式；不要保留不稳定的自定义刷新率。

3. **240Hz 生效后，单独测试垂直同步**
   - 在游戏内先测试 **垂直同步关闭**；不要同时修改帧率上限和其他延迟设置。
   - 依据等级：当前游戏设置的本机 A/B 实测。
   - 适用条件：目标是降低竞技输入延迟，并能接受可能出现的画面撕裂。
   - 预期观察：输入延迟下降；若撕裂明显，则记录结果后再恢复比较。
   - 回滚：重新开启垂直同步。

4. **再单独测试 NVIDIA Reflex**
   - 在 VALORANT 游戏内将 **NVIDIA Reflex 低延迟设为“开启”**，分别与“关闭”做同场景比较。
   - 暂时不要同时叠加 NVIDIA 控制面板中的低延迟模式。
   - “开启 + Boost”可能增加游戏本功耗、温度和风扇噪声，只在普通“开启”效果不佳且温度正常时再单独测试。
   - 依据等级：官方/一手，NVIDIA；具体收益依赖机器是否受 GPU 队列或功耗限制。
   - 适用条件：游戏内存在 Reflex 选项。
   - 预期观察：操作到画面反馈更直接，FPS 不一定提高。
   - 回滚：恢复原 Reflex 设置。

5. **最后处理帧率稳定性，而不是先追求超过240 FPS**
   - 240Hz 屏幕搭配约 220 FPS 仍然有价值，不必先升级硬件。
   - 比较“不限帧”和一个机器在激烈场景中能稳定维持的上限；不要直接套用所谓通用的 237、240 或 300 FPS。
   - 同时确认游戏使用 RTX 4060，并观察连续游戏十分钟后 1% low、帧时间、温度和功耗是否恶化。
   - 依据等级：本机 A/B 实测。
   - 适用条件：刷新率改为 240Hz 后仍有瞬时不跟手。
   - 预期观察：合理限帧可能改善帧时间稳定和温度；如果没有可重复收益，就保持原值。
   - 回滚：恢复原帧率上限和显卡选择设置。

## 暂不建议

- 不要为 FPS 关闭、删除或绕过 ACE。
- 不要把关闭 HVCI、VBS、Hyper-V、Secure Boot 或 TPM 当作常规优化。
- 不要使用来历不明的注册表、`bcdedit`、计时器、服务精简或“一键电竞优化包”。
- 不要先刷 BIOS、重装系统或反作弊；当前已有一个更直接且可验证的 60Hz 问题。
- 不要寻找 Raw Input Buffer 开关：该开关已在 11.06 移除并默认常开。
- 不要在 Windows 不提供 240Hz 时强行超频屏幕。

## 复测协议

在同一训练场或自定义场景、相同画质和后台应用下，每组测试至少三轮：

1. 记录 60Hz 基线。
2. 只改为 240Hz后复测。
3. 再分别单独测试垂直同步和 Reflex。

记录：

- avg FPS
- 1% low FPS
- 帧时间及尖峰
- 显示器实际刷新率
- RTT和丢包
- 连续运行十分钟后的体感

第一项成功标准不是 FPS 从 220 上升，而是 Windows 确认 **240Hz 已实际生效**，同时转视角和鼠标反馈明显改善。
