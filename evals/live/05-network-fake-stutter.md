# 诊断方案

## 结论与置信度

最可能是**晚间网络链路异常**，不是显卡性能问题；置信度：**高**。300 FPS 和帧时间稳定，同时出现 ping 20→100、丢包和角色回弹，是典型网络假卡顿。

**不用为此降画质，也不要专门关闭 HAGS。**二者不能修复 RTT 跳变或丢包。

## 已知 / 缺失

- 已知：国服、约 300 FPS、帧时间稳定；晚间延迟和丢包同步恶化。
- 推断：可能涉及 Wi‑Fi 干扰、家庭网络拥塞、运营商晚高峰路由或游戏线路。
- 仍缺：有线还是 Wi‑Fi、丢包方向、是否所有设备都受影响、热点对照结果。

## 症状分类

- 客户端 FPS：正常。
- 网络：异常，主路径。
- 显示/输入：没有直接异常证据。
- 反作弊：没有相关弹框或错误码，不进入 ACE/HVCI/VBS 排查。

## 优先动作

1. **同步显示网络与性能统计**
   - 依据等级：官方/一手。
   - 条件：在晚间复现时。
   - 观察：FPS、帧时间保持稳定，而 RTT/丢包与回弹同步出现。
   - 回滚：只读显示项，测完可关闭。

2. **如果目前使用 Wi‑Fi，短测一次网线直连**
   - 依据等级：区分实验。
   - 条件：电脑能够临时连接路由器。
   - 观察：若网线下丢包消失，优先排查 Wi‑Fi 干扰、信号或路由器负载；若仍异常，继续查上游线路。
   - 回滚：拔掉网线，恢复原连接。

3. **用手机热点进行一局短时对照**
   - 依据等级：区分实验，不代表热点长期更快。
   - 条件：手机流量和信号允许。
   - 观察：热点稳定而家庭宽带异常，更像本地网络或运营商线路；两者同样异常，才进一步考虑游戏节点或更广泛故障。
   - 回滚：切回家庭网络。

4. **排除家庭网络晚间占用**
   - 依据等级：可验证假设。
   - 条件：回弹发生时。
   - 操作：暂时停止云盘同步、直播、下载、系统更新和其他设备的大流量上传，然后复测。
   - 观察：ping 波动和丢包是否立即下降。
   - 回滚：恢复暂停的任务。

5. **连续记录三晚，形成可提交证据**
   - 依据等级：测量基线。
   - 条件：问题具有明显时间规律。
   - 记录：日期、时间、网络类型、服务器、RTT 中位数/峰值、丢包、回弹时间点。
   - 观察：若只在固定晚高峰出现，可将记录交给腾讯客服或宽带运营商定位线路。
   - 回滚：无系统改动。

## 暂不建议

- **降画质**：只会减轻 CPU/GPU 压力；当前 FPS 与帧时间已经稳定，无法解决网络丢包。
- **关闭 HAGS**：HAGS 属于图形调度设置，没有证据表明它能修复晚间 RTT 跳变。保持当前值，避免把网络诊断和图形设置混在一起。
- **关闭 HVCI/VBS、Hyper‑V 或 ACE 组件**：这些不是通用网络/FPS 优化项；只有出现腾讯指定的 CPU 虚拟化弹框时，才进入对应排查。
- **立即购买加速器**：可作为后续 A/B 对照，但不能仅凭广告宣称判断有效，更不能把 ping 下降误认为 FPS 提升。

## 复测协议

在同一时段、同一模式和同一网络条件下，每种连接至少测试三轮，记录：

- avg FPS、1% low、帧时间；
- RTT 中位数与峰值；
- 丢包率及回弹次数；
- Wi‑Fi、网线、手机热点之间的差异。

如果切换网络后 RTT/丢包明显变化，而 FPS/帧时间始终不变，即可基本确认是网络链路问题。
