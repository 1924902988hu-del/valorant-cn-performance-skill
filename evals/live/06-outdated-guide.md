# 诊断方案

## 结论与置信度

**不能原样照做。**截至 2026-07-22，这两条关键建议都已过时：Raw Input Buffer 已被移除并永久启用；VALORANT 也已加入游戏内 AMD Anti-Lag 2。**置信度：高。**

## 已知 / 缺失

- 已知：教程发布于 2024 年，要求打开 Raw Input Buffer，并声称 AMD 只能使用驱动级 Anti-Lag。
- 官方变化：
  - 11.06（2025-09-16）移除了 Raw Input Buffer 开关，功能始终启用。
  - 12.09（2026-05-12）加入 AMD Anti-Lag 2。
- 未知：你的 AMD 显卡型号、驱动版本、当前游戏补丁，以及游戏内是否显示 Anti-Lag 2。
- 未运行任何本机诊断；以下是设置判断，不代表已经完成优化。

## 症状分类

属于**显示与输入延迟链路**，不是网络或反作弊问题。

## 优先动作

1. **不要再寻找 Raw Input Buffer 开关。**  
   依据等级：官方/一手。  
   适用条件：当前版本客户端。  
   预期观察：游戏中没有该选项属于正常现象，原始输入缓冲已经常开。  
   回滚：无需回滚。

2. **如果游戏内出现 AMD Anti-Lag 2，优先测试游戏内选项。**  
   依据等级：官方/一手。  
   适用条件：兼容的 AMD 硬件，并满足官方驱动要求；参考资料归纳为使用 2026-03-09 之后所需的兼容驱动。  
   预期观察：操作响应可能改善，同时不应明显恶化帧时间或 1% low。厂商支持不等于你的电脑必然有收益。  
   回滚：恢复该游戏内选项的原值。

3. **不要同时叠加游戏内 Anti-Lag 2 和旧教程要求的驱动级 Anti-Lag。**  
   依据等级：官方功能事实＋可控 A/B 原则。  
   适用条件：游戏内 Anti-Lag 2 可用时。  
   预期观察：避免无法判断究竟哪个设置有效，以及潜在兼容或帧时间波动。  
   回滚：保持驱动设置原值，只测试一个低延迟路径。

4. **如果游戏内没有 Anti-Lag 2，先核对显卡和驱动兼容性。**  
   依据等级：官方/一手。  
   适用条件：选项缺失。  
   预期观察：确认是硬件、驱动还是客户端版本条件未满足。不要仅凭旧教程直接强开驱动 Anti-Lag。  
   回滚：如果后来升级驱动，事先记录原版本；发生回归时恢复原驱动。

5. **把整篇 2024 教程视为待复核资料。**  
   依据等级：官方补丁记录。  
   适用条件：教程还包含画质、HAGS、Game Mode、驱动面板或帧率上限建议。  
   预期观察：逐项验证，不能把“必开/必关”直接套到当前版本；游戏还在 11.02 从 Unreal 4.27 升到了 5.3。  
   回滚：无重复收益的设置恢复原值。

## 暂不建议

- 修改配置文件或注册表，试图重新显示 Raw Input Buffer。
- 因为教程写着“AMD 没有原生支持”就只使用驱动级 Anti-Lag——该说法自 12.09 起已经不成立。
- 同时修改 Anti-Lag、垂直同步、帧率上限和显示模式；这样无法定位效果。
- 把一次主观“更跟手”当成确定收益。

## 复测协议

保持相同地图/模式、分辨率、画质、帧率上限、后台程序和电源状态：

1. 记录原始设置，热身后测试至少三轮。
2. 记录 avg FPS、1% low 和帧时间波动。
3. 只切换游戏内 Anti-Lag 2，再重复三轮并比较中位结果。
4. 如果没有可重复收益，或 1% low、帧时间变差，就关闭并恢复原值。
5. 没有专用延迟测量设备时，“更跟手”只能作为主观辅助证据。
