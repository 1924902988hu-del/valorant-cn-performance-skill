# 诊断方案

## 结论与置信度

**不建议直接关闭“内存完整性”或 Hyper-V。**你没有 ACE 虚拟化弹框，平均 260 FPS 但周期性卡顿更像帧时间尖峰、后台任务或网络“假卡顿”，并非虚拟化冲突；置信度：**中**。

## 已知 / 缺失

- 已知：无畏契约国服、Windows 11 台式机、平均约 260 FPS、每隔几十秒卡顿、没有 ACE 虚拟化弹框。
- 缺失：卡顿瞬间的 FPS、帧时间、RTT、丢包、1% low，以及 CPU/GPU/内存、温度和后台程序数据。
- 未在你的 Windows 电脑上运行诊断，因此没有本机检测结果。

## 症状分类

目前属于**客户端瞬时卡顿与网络假卡顿待区分**。平均 FPS 无法反映几十毫秒的帧时间尖峰。

## 优先动作

1. **同时打开游戏内客户端 FPS、帧时间、网络 RTT 和丢包统计。**  
   依据等级：官方/一手。适用条件：复现卡顿时。预期观察：若 FPS 下跌、帧时间突增，是客户端路径；若 FPS/帧时间稳定但 RTT或丢包跳变，是网络路径。回滚：关闭统计显示即可。

2. **固定同一模式、分辨率、画质和帧率上限，热身后连续测试三轮。**  
   依据等级：测量规范。适用条件：所有后续实验。每轮记录 avg FPS、1% low、最大帧时间、RTT和丢包；不要只比较平均 FPS。回滚：无系统改动。

3. **关闭一个非必要后台项目再复测，例如浏览器、录制、Discord/NVIDIA 覆盖层或下载更新。**  
   依据等级：症状诊断。适用条件：卡顿具有几十秒一次的周期性。预期观察：帧时间尖峰频率或1% low是否可重复改善。回滚：重新打开对应程序；不要关闭 ACE、杀毒或 Windows 安全功能。

4. **观察卡顿瞬间的 CPU、GPU、内存、显存、温度和功耗。**  
   依据等级：硬件诊断。适用条件：客户端帧时间同时恶化。预期观察：是否出现温度/功耗撞墙、内存分页、显存压力或后台占用尖峰。回滚：只读观察，无需回滚。

5. **检查 Windows 实际刷新率。**  
   路径：`设置 → 系统 → 显示 → 高级显示`。依据等级：Microsoft 官方。适用条件：高 FPS 但体感不顺。预期观察：当前刷新率应等于显示器目标刷新率。若改动，回滚为原刷新率。

## 暂不建议

- **不提供关闭内存完整性或 Hyper-V 的步骤。**腾讯官方的相关排查只适用于出现“未开启或有其他软件占用 CPU 虚拟化功能”弹框，并且确认 BIOS 虚拟化设置后仍无法解决的情况。
- 关闭内存完整性会降低 Windows 内核保护；Hyper-V 还可能影响 WSL2、Docker、虚拟机、Sandbox及企业安全策略。
- 当前没有产品方证据证明关闭它们能解决你这种周期性卡顿。即使平均 FPS变化，也必须用1% low和帧时间证明收益。
- 不要关闭或删除 ACE，也不要使用所谓“一键优化包”、反作弊屏蔽器、注册表批处理或关闭 Secure Boot/TPM 的教程。

## 复测协议

在同一场景、同一设置下测试三轮，每次只改变一个项目，比较：

```text
avg FPS：
1% low：
卡顿瞬间帧时间：
RTT：
丢包：
卡顿次数/5分钟：
```

没有稳定、可重复的改善就恢复原设置。
