同一句“卡顿”，可能对应四条完全不同的路径：平均帧率低、百分之一低帧波动、网络假卡顿，或者高帧率却不跟手。国服 ACE 和国际服 Vanguard 的安全条件，也不能混用。

玩家只需描述真实症状。比如：国服 Windows 十一，平均二百六十帧，但每隔几十秒卡一下。有人让我关闭内存完整性，可我没有看到 ACE 虚拟化弹框。

FrameGuard CN 不会直接堆设置。没有特定 ACE 弹框，就不进入关闭 HVCI、VBS 或 Hyper-V 的路径。它会先让玩家同时观察帧时间、百分之一低帧、网络延迟和丢包，区分客户端与网络问题。

每轮只改一个类别，用同一场景复测；没有稳定收益，就恢复原值。最终交付包含置信度、已知和缺失事实、最多五项动作、依据等级、观察指标、回滚方法、复测协议和来源日期。

项目已公开发布 v1.0.0 安装包，并完成六个真实 Agent 回归，二百一十二项契约检查全部通过。这些证据验证路由、安全边界和交付结构，但不冒充 Windows 真机性能增益。

现在，FrameGuard CN 通过 OKX.AI 提供免费的“国服性能诊断”协作服务：从一句卡顿开始，交付一份可测、可回滚、不会诱导关闭安全能力的诊断方案。
