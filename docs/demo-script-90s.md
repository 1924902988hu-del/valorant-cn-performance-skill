# 80 秒演示脚本

## 0–10 秒：问题

屏幕放两条互相冲突的建议：“国服 ACE 必须关内存完整性”和“Vanguard On-Demand 需要 VBS/HVCI”。旁白：同一个 VALORANT，国服与国际服的反作弊路径不同；旧攻略还会让玩家寻找已经移除的 Raw Input Buffer。

## 10–22 秒：真实输入

给 Agent：

> 国服，Windows 11，平均 260 FPS 但每隔几十秒卡一下；有人让我关内存完整性。没有 ACE 虚拟化弹框。请帮我优化。

可选展示只读 `diagnose.ps1` JSON：版本、刷新率、VBS/HVCI、反作弊线索；强调不收集 IP/MAC/账号，不改系统。

## 22–46 秒：Skill 路由

突出三点：

1. 没有特定 ACE 弹框，所以不进入关闭 HVCI/VBS/Hyper-V 路径。
2. 先用帧时间与网络图表区分客户端 1% low 和网络假卡顿。
3. 每轮只改一类，用同一场景 avg FPS + 1% low 复测，无收益就回滚。

## 46–62 秒：交付

展示固定交付结构：置信度、已知/缺失、最多 5 项动作、暂不建议、复测协议、来源日期。明确写“诊断方案”，不是“优化完成”。

## 62–72 秒：可核验证据

展示公开 GitHub 仓库、v1.0.0 安装包、6 个真实 Agent 输出和 `212/212` 回归通过。旁白：这些验证覆盖路由、安全边界、当前版本事实和回滚结构，不把它们冒充 Windows 真机性能提升。

## 72–80 秒：OKX.AI

展示 FrameGuard CN 的“国服性能诊断”A2A 服务和 Agent ID。结尾：从一句“卡顿”开始，交付一份可测、可回滚、不会诱导关闭安全能力的诊断方案。

> 录制前必须把 Agent ID 占位替换为 OKX.AI 实际返回值；若仍在审核中，不得声称已经上线。
